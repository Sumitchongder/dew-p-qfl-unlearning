# Methodology Reference

This document specifies the exact formulas and implementation choices used
by `src/qflewp/`, for readers who want to verify or extend the code without
reading the full manuscript. It is written as software documentation
(what the code computes and why), not as a paper narrative — see the
manuscript for motivation, related work, and discussion.

## 1. Circuit model (`circuit.py`)

A data re-uploading variational quantum classifier on `n_qubits` qubits with
`n_layers` repeated blocks:

```
for layer in 0 .. n_layers-1:
    RX(scale * x_i)   on every qubit i      # data encoding, non-trainable
    RY(theta_k)       on every qubit        # trainable
    ring-CNOT(q, q+1) for q in 0..n_qubits-2, then CNOT(n_qubits-1, 0)
    RZ(theta_k)       on every qubit        # trainable
```

`n_trainable_params = n_qubits * 2 * n_layers`. The default `scale = 0.5`
radians was selected empirically (see `docs/METHODOLOGY.md#7`) to avoid the
encoding aliasing around the Bloch sphere for feature magnitudes beyond
roughly `[-2, 2]`.

State evolution is implemented directly in NumPy (`apply_1q`, `apply_cnot`
in `circuit.py`) rather than via a general-purpose simulator, batched over
the full sample batch simultaneously. This is mathematically identical to
simulating the equivalent Qiskit circuit (see `q1_deliverables.fig02_circuit_architecture`
for the Qiskit rendering used only for the architecture figure) but is
substantially faster for repeated parameter-shift evaluations at this qubit
count.

Prediction: `<Z_0>` expectation value is mapped to a class probability via
`p = (1 + <Z_0>) / 2`.

## 2. Federated training (`federated.py`)

Standard FedAvg: each client optimizes a local copy of `theta` on its own
data via `scipy.optimize.minimize(method="L-BFGS-B")`, using the *exact*
parameter-shift gradient of the binary cross-entropy loss (valid because
every trainable gate is a rotation generated by a Pauli operator with
eigenvalues ±1/2):

```
d<Z_0>/dtheta_k = 0.5 * ( f(theta + (pi/2) e_k) - f(theta - (pi/2) e_k) )
```

where `f` is the batched forward pass. The server aggregates local optima
by a sample-count-weighted average, matching the original FedAvg
aggregation rule (McMahan et al., 2017).

## 3. Diagonal Quantum Fisher Information (`qfim.py`)

For parameter `k` and a client `j`'s data `D_j`, the diagonal QFIM entry is
estimated via the parameter-shift / state-overlap identity (Stokes et al.,
"Quantum Natural Gradient", *Quantum* 4, 269 (2020)):

```
F_kk(x) = ( 1 - |<psi(theta; x) | psi(theta + pi * e_k; x)>|^2 ) / 4
F_kk^(j) = mean_{x in D_j} F_kk(x)
```

This is exact, not a finite-difference approximation, and — because the
circuit re-encodes `x` at every layer — genuinely conditions on the
forgotten client's data distribution, which is what the pruning score
requires.

## 4. Entanglement weight (`entanglement.py`)

For a parameter `k` on qubit `q` in layer `l`, `w_ent(k)` is the single-qubit
von Neumann entropy of qubit `q`'s reduced density matrix, evaluated
immediately after layer `l`'s ring-CNOT block (i.e. at the point in the
circuit where that layer's entanglement has just been generated), averaged
over a representative client-data batch:

```
rho_q = Tr_{not q}( |psi><psi| )
w_ent(k) = -sum_i lambda_i log2(lambda_i)     # eigenvalues of rho_q
```

Both the `RY` parameter (applied before the entangler) and the `RZ`
parameter (applied after) for qubit `q` in layer `l` receive the same
weight, reflecting that both gates shape the amplitude/phase feeding into
(or shaped by) that layer's entangling operation on that qubit.

## 5. Pruning score and unlearning methods (`pruning.py`)

```
s_k = w_ent(k) * F_kk^(j)
```

Parameters with `s_k <= tau` are pruned by setting them to exactly zero,
which is equivalent to replacing the gate with the identity (`RY(0) = RZ(0)
= I`). `tau` is chosen to hit a target pruning fraction (`prune_fraction`
in the config), not a fixed absolute value, so it is comparable across
random seeds and dataset draws.

Five baselines share the same `prune_by_score` mechanism with different
score vectors, plus two non-pruning baselines:

| Method | Score vector | Notes |
|---|---|---|
| `ewp_pruning` | `w_ent * F_kk` | proposed method |
| `fisher_only_pruning` | `F_kk` | isolates the QFIM signal |
| `entanglement_only_pruning` | `w_ent` | isolates the entanglement signal |
| `random_pruning` | uniform random, seeded | naive control |
| `fine_tune_only` | — (no pruning) | a few extra FedAvg rounds on retained clients |
| `full_retrain_oracle` | — (no pruning) | FedAvg trained from scratch, excluded client never participates |

An optional `recovery_fine_tune` step re-optimizes only the *surviving*
(non-pruned) parameters on retained-client data after pruning, keeping
pruned parameters frozen at exactly zero.

## 6. Evaluation suite (`evaluate.py`)

- **Utility**: accuracy and AUROC on retained-client test data, computed
  from the model's actual `predict_proba` output — not resampled or
  simulated.
- **Forgetting / privacy**: a confidence-thresholding membership-inference
  attack. The attacker sees prediction-confidence scores
  (`|p - 0.5| * 2`) on (a) the forgotten client's training samples
  ("members") and (b) held-out samples from clients that genuinely never
  contributed to training ("non-members"), and is scored by
  `membership_advantage = 2 * (AUC - 0.5)` (Yeom et al., 2018 style).
  `forgetting_score = 1 - |membership_advantage|`, so 1.0 means the
  attacker performs at chance.
- **Retrain distance**: `||theta_method - theta_oracle||_2`, the standard
  parameter-space proxy for how close an unlearning method is to exact
  unlearning (a genuine from-scratch retrain excluding the forgotten
  client).

## 7. Configuration schema

All experiment hyperparameters live in a single flat `dict` (see
`DEFAULTS` in `src/qflewp/pipeline.py`), passed explicitly through every
function rather than read from global state:

```python
{
    "n_qubits": int, "n_layers": int, "n_features": int,
    "n_clients": int, "samples_per_client": int,
    "non_iid_strength": float,   # client covariate-shift magnitude
    "noise": float,              # label-generating logistic noise std
    "n_rounds": int,             # FedAvg communication rounds
    "local_maxiter": int,        # L-BFGS-B iterations per client per round
    "forget_client": int,        # client ID targeted by the forget request
    "prune_fraction": float,     # tau, expressed as a target fraction
}
```

Every script in `scripts/` exposes these as CLI flags; no hidden
configuration files or environment variables are required to reproduce a
run.

## 8. Reproducibility notes

- All randomness is seeded explicitly (dataset generation seed =
  `1000 + run_seed`; the run seed also seeds model initialization, random
  pruning, and the membership-inference sample split).
- `results/json/main_experiment_raw.json` stores the full trained parameter
  vectors (`theta_full`, `theta_oracle`) and derived tensors (`qfim_diag`,
  `entanglement_weights`) for every seed, so `scripts/run_reconstruction.py`
  and `scripts/generate_deliverables.py` can be re-run without repeating
  the (comparatively expensive) federated training step.
- The only intentionally non-deterministic-across-machines component is
  wall-clock timing (`results/tables/table07_runtime_comparison.csv`),
  which depends on the host CPU; re-running `scripts/run_reconstruction.py`
  on different hardware will reproduce the same accuracy/forgetting/
  retrain-distance numbers but different absolute seconds.
